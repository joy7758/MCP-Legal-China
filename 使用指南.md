## 🛠️ 可用工具 (Tools)

### 1. check_contract_risk - 合同风险检查

**功能**: 自动检测合同中的法律风险

**参数**:
- `contract_text` (必需): 合同文本内容
- `check_types` (可选): 检查类型数组,可选值:
  - `jurisdiction` - 管辖权检查
  - `penalty` - 违约金检查
  - `liability` - 责任条款检查

**示例**:
```python
{
  "contract_text": "甲乙双方约定,争议提交纽约仲裁...",
  "check_types": ["jurisdiction", "penalty"]
}
```

**返回**:
```json
{
  "status": "发现风险",
  "risk_count": 2,
  "risks": [
    {
      "type": "jurisdiction",
      "level": "高风险",
      "description": "检测到非中国境内管辖权条款",
      "suggestion": "建议修改为: 北京仲裁委员会或上海仲裁委员会"
    }
  ]
}
```

---

### 2. analyze_legal_clause - 法律条款分析

**功能**: 分析特定条款的合规性

**参数**:
- `clause_text` (必需): 条款文本
- `clause_type` (必需): 条款类型
  - `penalty` - 违约金
  - `liability` - 责任
  - `termination` - 终止
  - `jurisdiction` - 管辖权

**示例**:
```python
{
  "clause_text": "违约方应支付合同总价20%的违约金",
  "clause_type": "penalty"
}
```

**返回**:
```json
{
  "clause_type": "penalty",
  "compliance_status": "基本合规",
  "legal_basis": [
    "《民法典》第585条 - 违约金条款"
  ],
  "suggestions": [
    "注意违约金比例,避免被认定为过高"
  ]
}
```

---

### 3. get_legal_suggestion - 获取法律建议

**功能**: 根据风险类型获取专业建议

**参数**:
- `risk_type` (必需): 风险类型
  - `jurisdiction` - 管辖权
  - `penalty` - 违约金
  - `liability` - 责任
  - `general` - 通用
- `context` (可选): 具体情况描述

**示例**:
```python
{
  "risk_type": "jurisdiction",
  "context": "涉及跨境电商业务"
}
```

**返回**:
```json
{
  "title": "管辖权条款建议",
  "recommendations": [
    "优先选择仲裁方式解决争议,效率更高",
    "建议选择: 北京仲裁委员会、上海仲裁委员会"
  ],
  "template": "因本合同引起的争议,提交北京仲裁委员会仲裁..."
}
```

## 🔧 配置说明

### 环境变量 (.env)

```bash

# 调试模式

API_RATE_PERIOD=60       # 时间窗口(秒)

PENALTY_THRESHOLD=0.3    # 30%
```

## 📖 使用场景

### 场景 1: 合同初步审查

```python
# 1. 使用 check_contract_risk 进行快速扫描
result = check_contract_risk({
    "contract_text": "...",
    "check_types": ["jurisdiction", "penalty", "liability"]
})

# 2. 如果发现风险,使用 analyze_legal_clause 深入分析
analysis = analyze_legal_clause({
    "clause_text": "...",
    "clause_type": "penalty"
})

# 3. 获取修改建议
suggestion = get_legal_suggestion({
    "risk_type": "penalty"
})
```

## 🐛 故障排除

### 问题 1: 服务器启动失败

**解决方案**:
```bash
# 检查虚拟环境
source venv/bin/activate

# 重新安装依赖
pip install -r requirements.txt

# 检查 Python 版本
python --version  # 需要 3.10+
```

---

### 问题 2: 配置加载失败

**解决方案**:
```bash
# 检查 .env 文件是否存在
ls -la .env

# 验证配置
python -c "from config import Config; print(Config.get_config_dict())"
```
